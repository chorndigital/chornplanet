# Internationalisation (i18n) Rules

## Supported Locales
The 10 supported locales are defined in `src/lib/SiteUrlLocales.ts`:
`da`, `de`, `en`, `fi`, `fr`, `ja`, `ko`, `nl`, `th`, `zh`

- `en` is the default locale.
- Every new piece of user-facing content MUST have a translation file for all 10 locales.

## Data / Translation Pattern
- Translation data lives in `src/data/` organised by feature (e.g. `src/data/ai/companions/fah/`).
- Each locale gets its own file: `AiFahEN.ts`, `AiFahTH.ts`, `AiFahFR.ts`, etc.
- A central `InfoTranslation` map (in `src/data/info/main/InfoTranslation.ts`) aggregates all locale data and is the single source of truth consumed by components.
- Components receive `lang: string` as a prop and index into `InfoTranslation[lang]`.

## Adding New Content
When adding new translatable content:
1. Create one file per locale in the appropriate `src/data/` subdirectory.
2. Export a typed object (define the type in a shared `types.ts` if reused).
3. Register each locale in the central `InfoTranslation` map.
4. Pass `lang` down from the page (read via `headers15.get('x-locale') || 'en'`).

## Metadata
- Page metadata lives in `src/metadata/main/Metadata<PageName>.ts`.
- Each metadata file exports a record keyed by locale: `Record<string, Metadata>`.
- Always provide metadata for all 10 locales.

## Do NOT
- Hardcode English strings in components — always use `InfoTranslation[lang]`.
- Add a new locale not in `LOCALES` without also updating `proxy.ts` matcher and `SiteUrlLocales.ts`.
