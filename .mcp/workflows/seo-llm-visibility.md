# SEO and LLM Visibility Workflow

Use this workflow when ChornPlanet work is intended to improve search discoverability, AI assistant understanding, LLM visibility, and structured platform explanation.

## Goal

Make ChornPlanet easier for search engines, social platforms, and AI assistants to understand.

## Target assets

- page metadata
- structured content pages
- topic clusters
- sitemap strategy
- public platform overview
- `llm.txt`
- `llm-full.txt`

## Process

1. Identify page, feature, or content category.
2. Review source-of-truth content.
3. Define target audience and intent.
4. Improve metadata and structured summary.
5. Add or update LLM-facing public summary if appropriate.
6. Validate that no secrets or private internal details are exposed.

## LLM files

`llm.txt` should be a concise public guide to ChornPlanet.

`llm-full.txt` can be a richer public guide with platform sections, content categories, commerce direction, and navigation guidance.

## Rules

- Do not expose secrets.
- Do not expose private repo/internal implementation details.
- Keep public claims accurate.
- Do not keyword-stuff.
- Prefer clear structure over excessive marketing language.

## Output artifacts

- metadata recommendation
- topic cluster plan
- `llm.txt` draft
- `llm-full.txt` draft
- AI discoverability review
