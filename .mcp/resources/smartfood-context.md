# Smart Food Context

ChornPlanet can become the web evolution layer for the Smart Food system.

## Smart Food direction

Future ChornPlanet features may support:

- Line OA user login
- food ordering via website
- favorite meal history
- customer meal preferences
- booking or reservation ideas
- food discovery and recommendations
- Smart Food product storytelling
- customer self-service experiences

## Login direction

Possible identity phases:

1. Line OA / LINE Login for Smart Food users.
2. Firebase Authentication in a later phase.
3. Unified profile experience when architecture is ready.

## Smart Food product flow

Potential future flow:

```text
Line OA user
  ↓
ChornPlanet login / profile
  ↓
favorite meals / previous orders / recommendations
  ↓
food order via website
  ↓
Smart Food backend order flow
  ↓
analytics and retention insight
```

## Safety rule

Food ordering, customer profile, payment, and order mutation workflows are high-risk.

Agents may plan these systems and generate implementation proposals.

Agents must not mutate live customer, order, payment, or identity data without explicit approval.

## Product-growth angle

Smart Food integration should help ChornPlanet increase:

- member registrations
- repeat visits
- food orders
- user personalization
- retention
- commerce credibility
- investor attractiveness
